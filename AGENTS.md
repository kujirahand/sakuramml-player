# サクラMIDIプレイヤー

ブラウザで動作するMIDI/MMLプレイヤーを作ります。
ブラウザでピアノロールを描画しながら、曲を演奏します。
UI と描画は JavaScript、MIDI解析や発音エンジンは Rust という分担にします。

## 全体の指示

- コーディングしていて学んだことは、docsフォルダにmdファイルとして追記して

- www/ --- HTML/JSのソースコード
  - fonts/ --- サウンドフォント

## 技術構成

Web Audio APIを使って音楽を再生します。
ブラウザ上で Rust を WebAssembly にして、音声生成の中核を Rust に持たせます。

Rust でまとめて PCM を生成し、JS 側で鳴らす方式で、Rust が Float32 の波形バッファをまとめて作り、JS 側がそれを AudioBuffer などに流し込んで再生します。

AudioWorklet でリアルタイム音声処理を行い、その中核計算を Rust/Wasm で行います。ただし、ローカルPCでも、ネイティブ再生できる仕組みも別途用意します。

AudioWorkletProcessor はレンダリングスレッドで動くので、リアルタイム DSP やシンセ処理に向いています。設計としては、JS の AudioWorklet 側が出力バッファを受け取り、Rust/Wasm に「このブロックを埋めて」と渡して処理させる形になります。1サンプルごとではなく、128フレーム単位以上の塊で処理するようにします。

### ノート列や音源状態は Rust 側に保持する

MIDIイベントの解釈、発音中ノートの管理、エンベロープ、発振器状態などを Rust 側に閉じ込め、JS は「再生開始」「停止」「テンポ変更」などの粗い命令だけ送るのが向いています。これは wasm-bindgen の「大きな長寿命データは Wasm 側に置く」という推奨とも一致します。
配列は TypedArray 前提でやり取りします。

## 各技術の役割分担

### Rust を使う部分

- MIDIパーサー
- ノート状態管理
- テンポ管理
- 音源

## JSを使う部分

- ファイル選択
- Canvas描画
- 再生ボタンやシークバー
- ピアノロールのズームとスクロール
- WebAudio ノードの接続

## 開発手順

- 1. Rust で MIDI を解析して、ノートのリストを出力するプログラムを作る
- 2. 簡単なPSG音源を Rust で実装して、Wasm にしてブラウザで鳴らす
- 3. HTML/JS でファイル選択と再生ボタンを作る
- 4. Rust の音源と MIDI パーサーを統合して、ブラウザで MIDI を再生できるようにする
- 5. ピアノロールを描画する
- 6. SoundFontを読み込んで、PCM波形を生成するようにする
- 7. (ハイブリッド化) BankSelect (CC#0) の値が 100 の場合は、元々の手作り PSG 音源で鳴らし、それ以外の場合は SoundFont を使って鳴らすようにする。


## メタテキストの描画

- MMLのコンパイルエラーなどがあれば、そこに表示する。
- MIDIイベントの歌詞があれば、テキストボックスの下のラベルに随時表示するようにする。
  - Marker/MetaTextイベントなどがあれば、ピアノロールの中にテキストを描画する
  - 描画位置は、小節番号の直下にする
